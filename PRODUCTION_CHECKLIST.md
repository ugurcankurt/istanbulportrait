# 🚀 Production Deployment Checklist

## 📋 Pre-Deployment Checklist

### 1. Code Quality & Testing
- [ ] **TypeScript compilation:** `npm run type-check`
- [ ] **Linting:** `npm run lint`
- [ ] **Build test:** `npm run build`
- [ ] **Production test suite:** `npm run production-test`
- [ ] All components render correctly
- [ ] No console errors in development

### 2. Environment Configuration
- [ ] **Production environment variables set**
  - [ ] `NEXT_PUBLIC_SUPABASE_URL` (production)
  - [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY` (production)
  - [ ] `SUPABASE_SERVICE_KEY` (production)
  - [ ] `IYZICO_BASE_URL` (production API)
  - [ ] `IYZICO_API_KEY` (production)
  - [ ] `IYZICO_SECRET_KEY` (production)
  - [ ] `NEXT_PUBLIC_GOOGLE_ANALYTICS_ID`
  - [ ] `RESEND_API_KEY` (optional)

### 3. Database Setup
- [ ] **Production Supabase project created**
- [ ] **Database migration executed:** `001_initial_schema.sql`
- [ ] **RLS policies active and tested**
- [ ] **Test database connectivity:** `npm run test:db`
- [ ] **Default packages inserted** (essential, premium, luxury, rooftop)

### 4. Payment System
- [ ] **Iyzico merchant account approved**
- [ ] **Production API keys obtained**
- [ ] **Test payment flow:** `npm run test:payment`
- [ ] **Webhook URL configured** (post-deployment)

### 5. GitHub & Version Control
- [ ] **Repository created:** `https://github.com/YOUR_USERNAME/istanbulportrait`
- [ ] **All code committed and pushed**
- [ ] **Git tags for versioning:** `git tag v1.0.0`
- [ ] **.gitignore excludes sensitive files**

## 🌐 Deployment Steps

### 1. Vercel Project Setup
1. **Create New Project**
   - Go to [vercel.com/new](https://vercel.com/new)
   - Import from GitHub: `istanbulportrait`
   - Framework: Next.js (auto-detected)

2. **Build Configuration**
   ```
   Framework Preset: Next.js
   Root Directory: ./
   Build Command: npm run build
   Output Directory: .next (auto)
   Install Command: npm install
   Node.js Version: 18.x (or latest)
   ```

3. **Environment Variables**
   - Copy from `.env.example`
   - Add production values
   - **Important:** Use production Supabase & Iyzico credentials

### 2. Domain Configuration
- [ ] **Custom domain added:** `istanbulportrait.com`
- [ ] **DNS records configured** (A records or nameservers)
- [ ] **SSL certificate active** (auto-generated by Vercel)
- [ ] **WWW redirect working:** `www.istanbulportrait.com` → `istanbulportrait.com`

### 3. Production Services Setup

#### Supabase Production
- [ ] **Production project created**
- [ ] **Migration executed via SQL Editor**
- [ ] **API keys added to Vercel environment**
- [ ] **Database connection tested**

#### Iyzico Production  
- [ ] **Merchant account active**
- [ ] **Production API keys in Vercel**
- [ ] **Webhook URL updated:** `https://istanbulportrait.com/api/payment/webhook`

## 🚨 Incident Response Plan

### Immediate Issues (within 5 minutes)
1. **Site down:** Check Vercel deployment status
2. **API errors:** Verify environment variables  
3. **Database issues:** Check Supabase status
4. **Payment failures:** Verify Iyzico credentials

### Rollback Procedure
1. **Vercel:** Rollback to previous deployment
2. **Database:** Restore from backup if needed
3. **DNS:** Revert DNS changes
4. **Communication:** Update status page/social media

### Emergency Contacts
- **Technical:** Development team lead
- **Business:** Project owner
- **External:** Vercel support, Supabase support
